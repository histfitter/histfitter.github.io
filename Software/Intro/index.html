Introduction

A software framework for statistical data analysis, called HistFitter, is described in detail here.

HistFitter is written in Python and C++, the former being used for configuration and the latter for CPU-intensive calculations. Internally, HistFitter uses the software packages HistFactory and RooStats, which are based on RooFit and ROOT to construct parametric models and perform statistical tests of the data.

Features

HistFitter extends these tools in four key areas:

1. Starting from an object-oriented configuration, defined by users, in a coherent way the framework builds probability density functions that are automatically fitted to data and interpreted with statistical tests.

2. A key innovation of HistFitter is its design, which is rooted in core analysis strategies of particle physics. The concepts of control, signal and validation regions are woven into its very fabric. These are progressively treated with statistically rigorous built-in methods.

3. Being capable of working with multiple data models at once, HistFitter introduces an additional level of abstraction that allows for easy bookkeeping, manipulation and testing of large collections of signal hypotheses.

4. Finally, HistFitter provides a collection of tools to present results with publication-quality style through a simple command-line interface.

Package structure:

The HistFitter package consists of:
  • ./: contains setup scripts and README.

  • scripts/: contains the top-level script HistFitter.py used for most fitting/hypothesis-testing/plotting operations starting from a user-defined configuration file, as well as several useful scripts for presentation of results by means of table/plot production.

  • python/: contains the Python configuration classes used to process user-defined configuration files, as well as the book-keeping machinery for histograms.

  • src/: contains classes and functions for fitting/hypothesis-testing/plotting, with calls to appropriate RooFit/RooStats functions, as well as the make file for the C++ side of the package.

  • config/: contains only the HistFactory schema file, that is used to produce a RooWorkspace from the histograms, which is a persistent RooFit object to transport a likelihood containing both the model and the data.

  • analysis/: contains some examples of user-defined configuration files used in the tutorial.

  • macros/: contains examples for contour production macros (also used in the tutorial), as well as example of using the C++ side of HistFitter without using the Python side.

  • data/: empty by default, but here all the histograms produced from a user-defined configuration get saved inside a ROOT file.

  • results/: empty by default, but here all the RooWorkspaces produced from a user-defined configuration get saved inside a ROOT file, as well as the plots produced by calls to HistFitter.py.

  • lib/: contains shared libraries, in particular libHistFitter.so that gets created once the package is compiled.
